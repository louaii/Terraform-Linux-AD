Automated Backend Setup (Option B) — Step-by-Step
Step 1 — Start with a Local Backend

Initially, use local state so Terraform can deploy resources without a remote backend.

Example local terraform {} section (optional; default is local):

terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}


Your Terraform will store state in terraform.tfstate locally.

Step 2 — Create a Terraform Module for Backend Resources

Create a folder modules/s3_dynamodb_backend/ with:

main.tf:

resource "aws_s3_bucket" "tf_state" {
  bucket = var.bucket_name
  acl    = "private"
  versioning {
    enabled = true
  }
}

resource "aws_dynamodb_table" "tf_lock" {
  name         = var.dynamodb_table_name
  billing_mode = "PAY_PER_REQUEST"
  hash_key     = "LockID"

  attribute {
    name = "LockID"
    type = "S"
  }
}


variables.tf:

variable "bucket_name" {
  type = string
}

variable "dynamodb_table_name" {
  type = string
}


outputs.tf:

output "s3_bucket_name" {
  value = aws_s3_bucket.tf_state.bucket
}

output "dynamodb_table_name" {
  value = aws_dynamodb_table.tf_lock.name
}

Step 3 — Deploy the Backend Resources

Create a temporary envs/bootstrap/ folder:

module "backend" {
  source = "../../modules/s3_dynamodb_backend"

  bucket_name         = "tf-state-louay"
  dynamodb_table_name = "terraform-lock"
}


Run:

terraform init
terraform apply


✅ This creates the S3 bucket and DynamoDB table automatically.

Step 4 — Switch to Remote Backend

Add your backend.tf in the main environment (e.g., envs/dev/) pointing to the new bucket and table:

terraform {
  backend "s3" {
    bucket         = "tf-state-louay"
    key            = "terraform.tfstate"
    region         = "us-east-1"
    dynamodb_table = "terraform-lock"
    encrypt        = true
  }
}


Run terraform init with migration:

terraform init -migrate-state


Terraform will move the local state to the S3 bucket.

From now on, all future apply/plan commands use remote state with locking.
